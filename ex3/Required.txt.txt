For encoding configuration, I decided to fix Rainfall_mm on the X-axis and Yield_hg_ha on the Y-axis because, during lecture we learned to position there the single most effective visual channels, so it only makes sense to give the two most important numeric variables the strongest encoding, this is also why in the code they are hardcoded directly into px.scatter(x="Rainfall_mm", y="Yield_hg_ha") instead of being part of the flexible radio buttons. I then assigned Continent to color because hue is the best way to separate categorical groups visually, and with only a handful of continents the colors stay distinguishable, the code passes this through color= in the scatter call. 

For Item I chose shape because I saw a similar example of Pinguins in the lecture, in addition shape naturally groups nominal categories into visible clusters without suggesting any order or ranking, which is exactly what crop types are. Temperature gets size since a bigger marker naturally feels warmer to a viewer, and it maps directly to size= in the code. 

Finally, I deliberately left Pesticides as None not because it is not important, but because during lecture we were warned that using too many visual channels at once causes them to mask and interfere with each other, making the whole chart harder to read; instead, the range slider from Task 2 already lets users explore pesticide values interactively, which is a much cleaner solution.
